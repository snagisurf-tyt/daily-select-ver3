## 開発時の注意点

- アプリのホストIPは、127.0.0.1 や 0.0.0.0 でなく、localhost あるいは 固定IP を使うこと。
- Pythonの仮想環境構築にはuvを使用する。ライブラリの追加は`uv add [パッケージ名] --link-mode=copy`コマンドを使う。
- 環境変数や設定値 `.env`または`.env.local`で一元管理する。ソースコード内に直接書かないこと。
- フロントエンドとバックエンドのシステムは完全に独立設計する。両者はAPIを介して通信する。

## AI生成コードの品質チェックリスト（提案G）

AIが生成するコードには固有のアンチパターンがある。コードレビュー・実装時に以下を確認すること。

### アンチパターン1: ゴーストファイル

**説明**: 既存ファイルを修正するかわりに、類似の新規ファイルを作成する。

**症状**:
- `utils.ts` と `utils_new.ts` が共存している
- `handler.py` と `handler_v2.py` が共存している
- `_old`, `_backup`, `_copy` サフィックスのファイルが存在する

**対策**:
- ファイルを新規作成する前に `Glob` で類似ファイルを検索する
- 既存ファイルの修正を最優先にする

### アンチパターン2: TypeScript `any` の乱用

**症状**: 型安全性を犠牲にした `any` / `unknown` の多用

**対策**:
- `any` は原則禁止（`tsconfig.json` に `"noImplicitAny": true` を設定）
- 型が不明な場合は `unknown` を使い、型ガードで絞り込む

### アンチパターン3: コードの重複

**症状**: 同一・類似ロジックが複数箇所に散在する

**対策**:
- 実装前に既存コードを検索し、再利用可能な関数・クラスがないか確認する
- 3箇所以上で同じロジックが必要になった時点で共通化を検討する

### アンチパターン4: 過剰コメント

**症状**: コードが自明なのにコメントで冗長に説明している

**対策**:
- コメントはロジックが自明でない箇所のみ（「なぜ」を書く）
- 「何をしているか」はコード自体を読みやすくして表現する

### アンチパターン5: セキュリティ脆弱性

**症状**: SQLインジェクション、XSS、コマンドインジェクション、ハードコードされた認証情報

**対策**:
- ユーザー入力は必ずバリデーション・エスケープする
- 認証情報は `.env` で管理し、コードに直接書かない
- SQLはORMかプリペアドステートメントを使用する

## ドキュメント管理戦略（提案C）

### ドキュメントの腐敗リスク

記述的な説明文書は必然的にコードに追いつけず腐敗する。AIエージェントが腐敗した情報を真実と誤認することを防ぐ。

### フロントマター規則

`docs/` 配下の重要ドキュメント（product-requirements.md, functional-design.md, architecture.md 等）には以下のフロントマターを付与すること:

```yaml
---
last_updated: YYYY-MM-DD
git_commit: abc1234  # このドキュメントが参照するコードのコミットハッシュ
status: current | outdated | draft
---
```

### ドキュメント整合性チェック

`/review-docs` コマンド実行時に以下を確認する:

1. フロントマターの `last_updated` が3ヶ月以上前のドキュメントを警告
2. `git_commit` が現在の `main` ブランチから大きく乖離しているドキュメントを報告
3. コードに実装されているが、ドキュメントに記載のない機能を検出

### 「実行可能アーティファクト」優先の原則

説明文書よりも以下を優先する:

| 代替手段 | 対象情報 |
|---------|---------|
| 型定義・スキーマ | データ構造・APIインターフェース |
| テストコード | 期待される動作・ユースケース |
| OpenAPI仕様 | APIエンドポイント・リクエスト/レスポンス |
| CI/CDパイプライン | ビルド・デプロイ手順 |

説明ドキュメントが「実行可能アーティファクト」と重複する場合、ドキュメントをポインタ（参照先を示すだけ）に縮小する。
