# 要求内容

## 概要

`testing` スキルおよび `add-feature-ui`/`add-feature` コマンドの動作確認ステップを強化し、アプリ起動確認・コンソールエラー確認・ネットワークエラー確認・E2E テストへのアサーション追加を実現する。

## 背景

`/add-feature-ui` で実装した機能について、ステップ7.5「動作確認・リグレッションチェック」が不十分なケースが発生している。playwright-cli による操作は行われているものの、各インタラクション後のコンソールエラー確認が省略されており、ボタン押下などの操作時に発生するランタイムエラーを見落とす問題がある。

## 実装対象の機能

### 1. testing スキルの全面書き直し
- チェックリスト形式から手順書（必須ステップ）形式に変換
- アプリ起動コマンドの自動検出・バックグラウンド起動フローを追加
- 「操作 → `playwright-cli console error` → `playwright-cli network` 確認」の必須サイクルを追加

### 2. add-feature-ui / add-feature ステップ7.5 の強化
- `Skill('testing')` 呼び出し前にコンソールエラー・ネットワークエラー確認が省略禁止である旨を明示

### 3. E2E テストコードへのアサーション追加指示
- add-feature-ui のステップ4（計画フェーズ）で E2E テストタスクにコンソールエラー・ネットワークエラーアサーションを含める注記を追加
- add-feature-ui のステップ5（実装ループ）でアサーションの具体的なコード例を提示

## 受け入れ条件

### testing スキル
- [ ] チェックリストではなく手順書として記述されている
- [ ] アプリ起動コマンドを `package.json` / `Makefile` / `README.md` から特定する手順が含まれる
- [ ] 各インタラクション後に `playwright-cli console error` を実行する必須サイクルが明記されている
- [ ] 各インタラクション後に `playwright-cli network` で 4xx/5xx を確認する手順が含まれる
- [ ] エラー検出時に作業を中断して `tasklist.md` に修正タスクを追加する指示がある

### add-feature-ui / add-feature ステップ7.5
- [ ] コンソールエラー確認・ネットワークエラー確認が「省略禁止」として明記されている

### E2E テストコード
- [ ] add-feature-ui ステップ4 の tasklist.md 生成指示に「コンソールエラーアサーションを含めること」の注記がある
- [ ] add-feature-ui ステップ5 に `page.on('console', ...)` と `page.on('response', ...)` のコード例がある

## スコープ外

- E2E テストコード自体の生成・変更（あくまでコマンドへの指示追加）
- 既存プロジェクトの E2E テストコードの修正
- `wait-on` パッケージのインストール自動化
