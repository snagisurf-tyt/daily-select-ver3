# 設計書

## アーキテクチャ概要

本機能は `.claude/commands/generate-readme.md` （Claude Code のカスタムコマンド定義ファイル）を編集するのみ。ソースコード（`src/`）の変更は不要。

```
generate-readme.md（変更対象）
├── ステップ1: 情報収集（既存）
├── ステップ1.5: 変更履歴の収集 ← 新規追加
├── ステップ2: スクリーンショット撮影の確認（既存）
├── ステップ3: バッジの生成（既存）
├── ステップ4: README.md の生成（既存 → テンプレート拡張）
│   └── 変更履歴セクション ← テンプレートに追加
│   └── タイムスタンプコメント ← テンプレートに追加
├── ステップ5: 既存README.mdの処理（既存）
└── ステップ6: 完了報告（既存 → 報告項目追加）
```

## コンポーネント設計

### 1. ステップ1.5: 変更履歴の収集（新規ステップ）

**責務**:
- README.md から前回生成日時（HTMLコメント）を読み取る
- git log から前回以降のコミットログを取得する
- コミットログを機能単位でグルーピング・要約する

**実装の要点**:
- git リポジトリチェック: `git rev-parse --is-inside-work-tree 2>/dev/null` で判定
- タイムスタンプ読み取り: README.md 内の `<!-- readme-generated: ... -->` を検索
- git log 取得: `git log --since="<日時>" --oneline --no-merges` （最大50件）
- 初回実行時（コメントなし）: 変更履歴は「初期リリース」のみ
- 非 git 環境: 変更履歴セクションを空で生成

### 2. テンプレート拡張（ステップ4の変更）

**責務**:
- README テンプレートに「変更履歴」セクションを追加
- 末尾にタイムスタンプ HTML コメントを付与

**実装の要点**:
- 変更履歴セクションの位置: 「ライセンス」セクションの直前
- 最新5件: テーブル形式で表示
- 6件目以降: `<details><summary>過去の変更履歴</summary>` で折りたたみ
- タイムスタンプ: `<!-- readme-generated: YYYY-MM-DDTHH:MM:SS -->` を末尾に付与

### 3. 完了報告の拡張（ステップ6の変更）

**責務**:
- 変更履歴セクションの生成状況を報告に追加

## データフロー

### 変更履歴の収集・生成フロー
```
1. README.md を読み込み、`<!-- readme-generated: ... -->` コメントを検索
2. コメントがあれば日時を取得、なければ初回実行とみなす
3. git rev-parse で git リポジトリか確認
4. git リポジトリの場合: git log --since="<日時>" --oneline --no-merges で差分取得（最大50件）
5. コミットログを解析し、機能単位でグルーピング・要約
   - feat: → 機能追加
   - fix: → バグ修正
   - 同一機能の複数コミット → 1行にまとめる
6. .steering/ 配下の requirements.md があれば機能名の参考にする
7. 要約結果を「日付 + 変更内容」のテーブル形式に変換
8. README テンプレートの変更履歴セクションに埋め込み
```

### 既存変更履歴との統合フロー
```
1. README に既存の変更履歴セクションがある場合、既存エントリを抽出
2. 新しいエントリを先頭に追加
3. 重複エントリ（同一日付・同一内容）を除去
4. 最新5件をテーブルに、6件目以降を折りたたみに分割
```

## エラーハンドリング戦略

### フォールバック
- git リポジトリでない場合: 変更履歴セクションをプレースホルダ付きで空生成
- コミットメッセージが `feat:` / `fix:` 形式でない場合: メッセージをそのまま使用
- git log が大量にある場合: 直近50件に制限

## テスト戦略

本機能はカスタムコマンド定義（Markdown）の変更のみであり、ソースコード変更がないため、ユニットテスト・統合テストの対象外。受け入れ条件は手動確認で検証する。

## 依存ライブラリ

新規ライブラリの追加なし。

## ディレクトリ構造

```
.claude/commands/
  generate-readme.md  ← 変更対象（唯一のファイル）
```

## 実装の順序

1. ステップ1.5（変更履歴の収集）をコマンドに追加
2. ステップ4 のテンプレートに変更履歴セクションとタイムスタンプを追加
3. ステップ5 の既存 README 処理に変更履歴統合ロジックを追加
4. ステップ6 の完了報告に変更履歴の項目を追加

## セキュリティ考慮事項

- git log コマンドは読み取り専用であり、セキュリティリスクなし

## パフォーマンス考慮事項

- git log の取得件数を最大50件に制限し、処理時間を抑制

## 将来の拡張性

- `/add-feature` 完了時の自動 README 更新への拡張が容易な構造
- CHANGELOG.md への分離も変更履歴生成ロジックの再利用で対応可能
