# 要求内容

## 概要

`/generate-readme` コマンドに変更履歴セクションの自動生成機能を追加する。git log の差分を機能単位で要約し、README.md に反映する。

## 背景

`/generate-readme` で README.md を生成した後、プロジェクトの変更履歴が README に反映されない。開発が進むたびに手動で更新するのは非現実的であり、コマンド実行時に git log から自動的に変更履歴を生成・更新する仕組みが必要。

## 実装対象の機能

### 1. 変更履歴の収集・生成ステップ追加
- `/generate-readme` のステップ1（情報収集）の後に、git log から変更履歴を収集するステップを挿入
- 前回生成日時以降のコミットログを取得し、機能単位で要約する
- git リポジトリでない場合はスキップ

### 2. README テンプレートへの変更履歴セクション追加
- 「ライセンス」セクションの直前に「変更履歴」セクションを配置
- 最新 5 件をテーブル形式で表示
- 6 件目以降は `<details>` タグで折りたたみ

### 3. タイムスタンプ管理
- README.md 末尾に `<!-- readme-generated: YYYY-MM-DDTHH:MM:SS -->` の HTML コメントを埋め込み
- 次回実行時にこのコメントを読み取り、差分取得の起点とする

## 受け入れ条件

### 変更履歴の生成
- [ ] `/generate-readme` 実行後の README.md に「変更履歴」セクションが含まれること
- [ ] 変更履歴が機能単位でグルーピング・要約されていること
- [ ] 最新 5 件が表示され、それ以前が折りたたまれていること

### タイムスタンプ管理
- [ ] README 末尾に `<!-- readme-generated: ... -->` コメントが存在すること
- [ ] 2 回目以降の実行で、前回以降の差分のみが追加されること

### 堅牢性
- [ ] git リポジトリでない環境でもエラーにならないこと

## 成功指標

- `/generate-readme` を実行するだけで変更履歴が自動生成される
- README の読者がプロジェクトの最近の変更点を一目で把握できる

## スコープ外

以下はこのフェーズでは実装しません:

- `/add-feature` 完了時の自動 README 更新
- 別ファイル（CHANGELOG.md）での管理
- Conventional Commits の厳密なパース
